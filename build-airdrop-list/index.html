Airdrop List Builder

Build CSV output for collection migrations using on-chain owner data.

Requires a wallet like Metamask to be installed. Owner type can either be Wallet or Contract. Delete the first row and third column before using on Scatter.

Requires ERC-721 contracts with ownerOf and totalSupply read methods. If your source contract starts at token ID 0, enable the toggle so IDs are shifted correctly for Scatter.

Use at your own risk and sanity-check results. If nothing outputs, check the browser console. If there are gaps in minting where some IDs were never minted, do not rely on this tool.

Expected sequence example: if owners are you(1), me(2,3), you(4), the list should read you,1 then me,2 then you,1 so migrated IDs remain aligned.
